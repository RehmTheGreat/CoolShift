DATA RESPONSIBILITY
- Organizer supplies 3 public scenarios, each covering 30 days at 15-minute resolution.
- Total public inputs: 8,640 interval records.
- Each team creates 1 custom seven-day scenario: 672 interval records.
- Required submitted outputs: 2,688 interval rows.
- Hidden evaluation: 1 unseen 24-hour case producing 96 recommendations.

The custom scenario may use API data, open data, manually structured realistic values, synthetic data, or a combination. Teams must disclose the source and make synthetic generation reproducible.

IMPORTANT
More rows or larger claimed savings do not prove optimization. Judges will verify baseline comparison, comfort and safety, energy balance, outages, battery constraints, appliance limits, dynamic recalculation and unseen-case performance.

NON-NEGOTIABLES
- No hard-coded outputs.
- No static awareness website, basic calculator or generic chatbot as the complete solution.
- Keep the original public workbook unchanged; work from a copy.
- Use the supplied output templates and preserve required column names.
- Disclose all APIs, datasets, models, libraries and reused components.
- Label features as functional, simulated or planned.
